## 6. Рефлексия
- Что было интересным: Наиболее интересным было увидеть, как отдельные части кода, написанные разными участниками, собираются в работающее приложение через Pull Requests. Процесс Code Review открыл новые аспекты командной работы.
- Что вызвало трудности: Наибольшие трудности возникли при разрешении конфликтов слияния в Git, особенно когда несколько человек работали над одними и теми же файлами. Также потребовалось время, чтобы привыкнуть к workflow через Issues и Pull Requests.
- Чему научились: Освоил полный цикл разработки ПО: от планирования задач в Issues до реализации, Code Review и слияния кода. Научился профессионально документировать проекты на Markdown, работать с ветками в Git, использовать IDE для эффективной разработки.
- Как планируете применять знания: Полученные навыки буду применять в учебных проектах, а в будущем - в профессиональной деятельности. Особенно ценен опыт командной работы через GitHub, который является стандартом в современной разработке.
